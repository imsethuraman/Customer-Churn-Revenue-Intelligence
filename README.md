# Customer-Churn-Revenue-Intelligence
SaaS Analytics Project (SQL + Power BI)
This project analyzes customer churn, engagement, and revenue patterns in a SaaS business using MySQL and Power BI.  The goal is to identify key churn drivers and provide data-driven recommendations to improve customer retention and business performance.
🎯 Business Problem
High customer churn is impacting revenue
Limited visibility into customer behavior
Need for actionable insights to improve retention

🗂️Dataset
The project uses a SaaS-style dataset with the following tables:
Accounts → Customer details
Subscriptions → Revenue & plans (MRR, ARR)
Feature Usage → Product engagement
Support Tickets → Customer issues
Churn Events → Churn tracking

⚙️ Tech Stack
🧮 MySQL → Data extraction, joins, CTEs
📊 Power BI → Dashboard & visualization
📈 DAX → KPI calculations & segmentation
🧠 Analytics → Business insights & storytelling

🔍 Key Analysis
Churn rate calculation
Revenue trend analysis (MRR, ARR)
Feature usage vs churn
Support ticket impact on churn

📈 Key Insights
⚠️ ~70% churn rate indicates a critical retention issue
📉 Low activity users churn the most
📞 High support tickets increase churn risk
⏳ Most churn occurs within the first 90 days
💰 High-value customers show better retention

🎯 Business Recommendations
🚀 Improve onboarding experience
📊 Increase feature adoption
⚡ Optimize support response time
🎯 Target at-risk customers

💼 Business Impact
Reduce churn by 15–25%
Increase customer lifetime value
Improve retention strategy
Enable data-driven decisions
